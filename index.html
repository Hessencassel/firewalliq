Security by design

Your config is processed in memory.
Nothing is ever stored.

When you paste a config and click Run Audit, it travels over HTTPS directly to a serverless function. That function passes it to the audit engine, streams the findings back to your screen, and terminates. No database writes. No file storage. No logging of your config. When the process ends, the data is gone.

In transit only

Your config exists only while the audit is running. The moment the serverless function completes, it is gone from memory entirely. There is no background job, no queue, and no cache holding it.

No database calls

There is no code path that writes your configuration to any database, cache, or storage layer. The function receives it, passes it to the audit engine, and discards it. You can read the source on GitHub.

HTTPS end to end

Every byte travels over TLS from your browser to the serverless function. Your config is never transmitted unencrypted at any point in the pipeline.

Skeptical? The architecture is simple enough to verify. No config has ever touched a database in FirewallIQ's codebase.
