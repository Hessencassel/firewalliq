Security & Privacy

How FirewallIQ handles your data

The short version: we don't store it. Here is exactly why that is true.

The audit pipeline

The user pastes a config and clicks Run Audit. The browser sends it over HTTPS to a Netlify serverless function. That function receives the config in memory and passes it directly to the Anthropic API as part of the audit prompt. The API streams findings back through the function to the user's browser. The function then terminates and its memory is cleared.

No step in this pipeline involves writing anything to disk, a database, a cache, or a log file.

What we do store

FirewallIQ stores three things and nothing else. Your account email address. Your plan type (Pro or Team). Your monthly audit count.

That is the complete list. We do not store config text, audit results, IP addresses mapped to configs, or any record of what you audited.

Why serverless makes this verifiable

FirewallIQ's audit engine runs as a Netlify serverless function. Serverless functions have no persistent memory between invocations. Every time the function runs, it starts from a clean slate. Even if we wanted to retain your config, the function would need to explicitly write it to an external service. It does not.

The relevant code is in audit.mjs in the public GitHub repository. The absence of any storage call is something you can verify yourself.

Your data rights

You can delete your account at any time by emailing support@firewalliq.io. Deletion removes your email address and plan record from our database. There is no config history to delete because none was ever created.

If you have questions about what data we hold, email us and we will respond within 48 hours.
